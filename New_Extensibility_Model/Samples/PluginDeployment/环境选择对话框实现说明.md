# 环境选择对话框实现说明
# Environment Selection Dialog Implementation Documentation

## 概述 (Overview)

本文档详细说明了 PluginDeployment 项目中环境选择对话框的实现细节。该功能允许用户在 Visual Studio 中通过多步骤向导配置 Dynamics CRM 插件发布的目标环境和认证方式。

This document provides detailed explanation of the environment selection dialog implementation in the PluginDeployment project. This feature allows users to configure target environment and authentication method for Dynamics CRM plugin deployment through a multi-step wizard in Visual Studio.

## 功能特性 (Features)

### 1. 环境类型选择 (Environment Type Selection)
- **On-Premise**: 本地部署的 CRM 环境
- **Dataverse**: 云端 Dynamics 365/Dataverse 环境

### 2. 认证方式选择 (Authentication Method Selection)
根据选择的环境类型，提供不同的认证选项：

**On-Premise 环境:**
- Active Directory
- IFD (Internet Facing Deployment)
- 连接字符串 (Connection String)

**Dataverse 环境:**
- OAuth
- 连接字符串 (Connection String)

### 3. 连接配置 (Connection Configuration)
- 连接字符串输入
- Dataverse URL 输入
- 输入验证和错误处理

## 技术实现 (Technical Implementation)

### 核心类和枚举 (Core Classes and Enums)

```csharp
// 环境类型枚举
internal enum CrmEnvironmentType
{
    OnPremise,    // On-Premise 环境
    Dataverse     // Dataverse 云端环境
}

// 认证方式枚举
internal enum AuthenticationMethod
{
    OAuth,              // OAuth 认证
    ConnectionString,   // 连接字符串
    ActiveDirectory,    // Active Directory
    IFD                 // Internet Facing Deployment
}
```

### 对话框流程 (Dialog Flow)

#### 第一步：环境选择
```csharp
CrmEnvironmentType selectedEnvironment = await shell.ShowPromptAsync(
    "请选择目标环境类型：",
    new PromptOptions<CrmEnvironmentType>
    {
        Choices = {
            { "On-Premise (本地部署)", CrmEnvironmentType.OnPremise },
            { "Dataverse (云端环境)", CrmEnvironmentType.Dataverse },
        },
        DismissedReturns = CrmEnvironmentType.OnPremise,
        Title = DialogTitle,
        Icon = ImageMoniker.KnownValues.Settings,
    },
    cancellationToken);
```

#### 第二步：认证方式选择
根据第一步选择的环境，显示相应的认证选项：

```csharp
// On-Premise 环境的认证选项
if (selectedEnvironment == CrmEnvironmentType.OnPremise)
{
    selectedAuth = await shell.ShowPromptAsync(
        "请选择 On-Premise 环境的认证方式：",
        new PromptOptions<AuthenticationMethod>
        {
            Choices = {
                { "Active Directory", AuthenticationMethod.ActiveDirectory },
                { "IFD (Internet Facing Deployment)", AuthenticationMethod.IFD },
                { "连接字符串", AuthenticationMethod.ConnectionString },
            },
            // ...
        },
        cancellationToken);
}
```

#### 第三步：连接详细信息
根据认证方式要求用户输入必要的连接信息：

```csharp
if (selectedAuth == AuthenticationMethod.ConnectionString)
{
    connectionDetails = await shell.ShowPromptAsync(
        "请输入连接字符串：",
        InputPromptOptions.Default with { 
            Title = DialogTitle,
            Icon = ImageMoniker.KnownValues.Database,
        },
        cancellationToken);
}
```

#### 第四步：配置确认
显示配置摘要并确认执行：

```csharp
string configSummary = $"环境类型：{(selectedEnvironment == CrmEnvironmentType.OnPremise ? "On-Premise" : "Dataverse")}\n" +
                       $"认证方式：{GetAuthMethodDisplayName(selectedAuth)}\n" +
                       $"连接信息：{(string.IsNullOrEmpty(connectionDetails) ? "使用默认配置" : "已配置")}";

bool confirmDeployment = await shell.ShowPromptAsync(
    $"确认发布配置：\n\n{configSummary}\n\n确定要继续发布插件吗？",
    PromptOptions.OKCancel with {
        Title = DialogTitle,
        Icon = ImageMoniker.KnownValues.StatusInformation,
    },
    cancellationToken);
```

## 用户体验设计 (User Experience Design)

### 1. 渐进式引导 (Progressive Guidance)
- 逐步收集用户配置，避免一次性展示过多选项
- 根据前面的选择动态调整后续选项

### 2. 智能默认值 (Smart Defaults)
- 提供合理的默认选择（如 OnPremise 作为默认环境）
- 允许用户快速确认常用配置

### 3. 错误处理 (Error Handling)
- 输入验证：确保必填字段不为空
- 友好的错误消息和取消机制
- 异常捕获和用户友好的错误提示

### 4. 本地化支持 (Localization Support)
- 完整的中文界面
- 扩展性强的字符串资源管理
