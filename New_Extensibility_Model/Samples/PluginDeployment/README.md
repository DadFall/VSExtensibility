# Dynamics CRM 插件发布工具（PluginDeployment）\n\n基于 Visual Studio 新扩展性模型（Non‑VSIX）的示例扩展，目标是简化 Dynamics 365/Dataverse 与 On‑Premise 环境中的插件程序集发布与部署。当前版本已在项目节点添加“插件发布”入口，后续将逐步完善环境选择与自动发布流程。\n\n## 当前进展\n- 已实现\n  - 解决方案资源管理器项目节点右键菜单：显示“插件发布”命令\n  - 异步执行与状态提示（示例提示“插件发布中”）\n  - 中文本地化（string-resources.json）\n- 进行中 / 计划中\n  - 环境选择对话框与多环境配置（On‑Premise 与 Dataverse）\n  - 认证方式：OAuth、连接字符串、AD、IFD\n  - 发布管道：自动构建、上传插件程序集，解析并同步 `[CrmPluginRegistration]` 步骤与映像\n  - 发布选项：目标解决方案唯一名、隔离模式、源类型等\n  - 日志与重试机制\n\n## 安装与运行\n- 在 Visual Studio 2022 中打开解决方案\n- 构建 PluginDeployment 项目，按 F5 启动调试\n- 扩展将自动部署到 Visual Studio 实验实例（基于新扩展性模型，非传统 VSIX 安装）\n\n## 使用\n1. 在解决方案资源管理器中右键任意项目\n2. 选择“插件发布”\n3. 当前版本会显示“插件发布中”提示；后续版本将弹出环境选择并执行发布\n\n## 技术概要\n- 目标框架：.NET 8.0 for Windows（net8.0-windows8.0）\n- 语言版本：C# 12；启用隐式 using 与可空引用类型\n- 依赖：\n  ```xml\n  <PackageReference Include="Microsoft.VisualStudio.Extensibility.Sdk" Version="17.14.40254" />\n  <PackageReference Include="Microsoft.VisualStudio.Extensibility.Build" Version="17.14.40254" />\n  ```\n- 菜单位置（VSCT）：\n  ```csharp\n  CommandPlacement.VsctParent(new Guid("{d309f791-903f-11d0-9efc-00a0c911004f}"), id: 518, priority: 0)\n  ```\n  - 518：项目节点右键菜单；常见 ID：537（解决方案），521（项目文件）\n- 本地化键：\n  ```json\n  {\n    "PluginDeployment.PluginDeployCommand.DisplayName": "插件发布",\n    "PluginDeployment.PluginDeployCommand.ProgressMessage": "插件发布中"\n  }\n  ```\n\n## 项目结构\n``````\nPluginDeployment/\n├─ .vsextension/\n│  └─ string-resources.json\n├─ ExtensionCommandConfiguration.cs\n├─ PluginDeployCommand.cs\n├─ PluginDeployment.csproj\n├─ PluginDeploymentExtension.cs\n└─ README.md\n``````\n\n## 兼容性\n- Visual Studio 2022 及更高版本\n- Windows 8.0 及更高版本\n- 新扩展性模型（Non‑VSIX）\n\n## 贡献\n1. **Fork 本项目**：从 GitHub 上 fork 本项目并进行修改。\n2. **提交 PR**：提交 pull request 以贡献代码或修复 bug。\n3. **报告问题**：在 Issues 页面提交 bug 或建议。\n\n## 联系方式\n\n如果你有任何问题或建议，请通过以下方式联系：\n\n- **邮件**：[guangzhu.zhu@foxmail.com]\n- **GitHub**：[https://github.com/DadFall/PluginPublisher](https://github.com/DadFall/PluginPublisher)\n\n## 许可证\n\n此项目使用 MIT 许可证。详情请参阅 LICENSE 文件。
